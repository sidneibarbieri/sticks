# Realism Notes

This repository distinguishes between canonical reproducibility and optional
substrate realism.

## Fidelity Model

Technique executions are classified as:

- `faithful`
- `adapted`
- `inspired`

These labels are methodology markers, not marketing terms. They should be read
from the generated evidence and the campaign metadata together.

## Practical Meaning

- The canonical smoke path demonstrates the repository-local execution model.
- The optional VM-backed path is where substrate realism is studied and measured.
- Windows-oriented procedures that are simplified or run on Linux should remain
  explicitly marked as `adapted` or `inspired`.

## Proposed Initial Activity: "Phishing Campaign with PowerShell Payload"

### Activity Objective
Execute a complete attack chain that demonstrates how an attacker can establish a foothold through phishing, deploy a PowerShell-based payload, establish persistence, and create a command and control channel.

### Activity Scope
This activity will implement the full attack chain from the 0.c0011 campaign in a controlled, linear sequence that can be executed and measured end-to-end.

### Prerequisites
1. **Target Environment:** Windows machine with PowerShell enabled
2. **Network Access:** Internet connectivity for C2 communication
3. **Privileges:** Standard user privileges (activity demonstrates privilege escalation potential)
4. **Tools:** Basic scripting capabilities and network tools

### Execution Sequence

#### Phase 1: Resource Preparation (Resource Development)
1. **Establish Accounts** (3ee16395-03f0-4690-a32e-69ce9ada0f9e)
   - Create a legitimate-looking email account for phishing
   - Set up a domain or subdomain for hosting malicious content

2. **Virtual Private Server Setup** (1cec9319-743b-4840-bb65-431547bce82a)
   - Provision a VPS for hosting payload and C2 infrastructure
   - Configure web server for payload delivery

#### Phase 2: Initial Compromise (Initial Access)
3. **Spearphishing Link** (2b742742-28c3-4e1b-bab7-8350d6300fa7)
   - Craft targeted phishing email with malicious link
   - Host obfuscated PowerShell script on VPS
   - Deliver email to target user

#### Phase 3: Payload Execution (Execution)
4. **PowerShell Execution** (232b7f21-adf9-4b42-b936-b9d6f7df856e)
   - Target clicks link, downloads and executes PowerShell script
   - Script performs initial reconnaissance and establishes foothold

5. **User Execution** (ef67e13e-5598-4adc-bdb2-998225874fa9)
   - Script prompts user for additional actions (social engineering)
   - User inadvertently executes additional malicious commands

#### Phase 4: Persistence and Evasion (Defense Evasion)
6. **Obfuscated Files/Information** (2e34237d-8574-43f6-aace-ae2915de8597)
   - Script downloads additional payloads with obfuscation
   - Encodes sensitive information and commands

7. **Modify Registry** (dfd7cc1d-e1d8-4394-a198-97c4cab8aa67)
   - Create registry entries for persistence
   - Modify system settings to maintain access

#### Phase 5: Command and Control (Command and Control)
8. **Web Protocols** (40f5caa0-4cb7-4117-89fc-d421bb493df3)
   - Establish communication with C2 server via HTTP/HTTPS
   - Set up bidirectional command channel
   - Begin data exfiltration preparation

### Expected Outputs
1. **Execution Log:** Detailed log of each technique execution
2. **Network Traffic:** Captured C2 communications
3. **System Changes:** Registry modifications and file system changes
4. **Payload Artifacts:** Created files and scripts
5. **Success Indicators:** Confirmation of each phase completion

### Validation Criteria
1. **Phase Completion:** Each of the 5 phases completes successfully
2. **Technique Execution:** All 8 techniques are executed as intended
3. **Chain Integrity:** Successful progression from initial access to C2 establishment
4. **Detection Points:** Clear indicators that can be used for defensive analysis
5. **Reversibility:** Activity can be safely cleaned up and reversed

### Implementation Notes
- This activity provides a complete, end-to-end attack simulation
- Each phase builds upon the previous one in a logical sequence
- The activity demonstrates real-world attack patterns in a controlled manner

- The linear flow makes it easy to implement, execute, and analyze

This activity serves as an excellent foundation for understanding attack chains and can be expanded with additional techniques or defensive countermeasures as needed.

# Campaign 0.c0011 Defensive Emulation Specification

## 1. Activity Title
**"Controlled ATT&CK Chain Emulation: Phishing to C2 Simulation"**

## 2. Purpose
This lab exercise is designed to validate defensive detection capabilities and ATT&CK mapping accuracy by simulating a complete attack chain in a controlled environment. The focus is on:

- **Telemetry Generation**: Capture comprehensive logs across the entire attack lifecycle
- **ATT&CK Mapping Validation**: Verify detection rules correctly identify each technique
- **Analytic Validation**: Test existing detection logic against known attack patterns
- **Response Workflow Testing**: Practice incident response procedures for multi-phase attacks
- **Blue Team Training**: Provide realistic but safe scenarios for SOC analysts

## 3. Scope and Safety Boundaries

### Lab Assumptions
- **Closed Environment**: All activities contained within isolated lab network
- **Instrumented Hosts**: Target systems have comprehensive logging enabled
- **Controlled Data**: No real user data or production systems involved
- **Benign Artifacts**: All files and scripts are safe, non-malicious placeholders

### Isolation Requirements
- **Network Segregation**: Lab environment completely isolated from production networks
- **Host Isolation**: Target machines have no access to external networks
- **Data Containment**: All generated artifacts remain within lab boundaries
- **No Real Phishing**: No actual email delivery or user interaction with malicious content

### Prohibited Actions
- ❌ Real payload development or deployment
- ❌ Actual phishing email creation or distribution
- ❌ Registry modifications that could persist beyond lab session
- ❌ Network communications outside controlled lab environment
- ❌ Obfuscation techniques that could be misused
- ❌ Persistence mechanisms that survive system restart

### Allowed Simulation Methods
- ✅ Benign PowerShell scripts that log their execution
- ✅ Mock HTTP requests to local test servers
- ✅ Registry read operations (no writes)
- ✅ Process creation with safe, documented scripts
- ✅ Network traffic to controlled endpoints
- ✅ File system operations with temporary, safe files

### Rollback and Cleanup
- **Automatic Cleanup**: All artifacts automatically removed after exercise
- **Host Reset**: Target systems restored to baseline state
- **Log Preservation**: Security logs preserved for analysis
- **No Persistent Changes**: Registry and file system returned to original state

## 6. Telemetry Requirements

### Windows Process Creation
**Required Fields**: Process name, command line, parent process, user context
**Evidence to Capture**: PowerShell execution, script launches, process relationships
**Setup**: Enable Windows Event Log 4688, Sysmon Process Create events

### PowerShell Logging
**Required Fields**: Script block text, execution policy, module loading
**Evidence to Capture**: All PowerShell activity, script content, execution context
**Setup**: Enable PowerShell transcription, module logging, script block logging

### Registry Auditing
**Required Fields**: Registry key path, operation type, process context
**Evidence to Capture**: Registry access patterns, modification attempts
**Setup**: Enable registry auditing policies, monitor key access

### DNS Logs
**Required Fields**: Query domain, response, timestamp, source IP
**Evidence to Capture**: Domain resolution patterns, suspicious lookups
**Setup**: Configure DNS server logging, forward to SIEM

### Proxy/HTTP Logs
**Required Fields**: URL, method, user agent, response code, timing
**Evidence to Capture**: Web protocol usage, C2 communication patterns
**Setup**: Enable proxy logging, capture full HTTP headers

### EDR Events
**Required Fields**: Process behavior, file operations, network connections
**Evidence to Capture**: Behavioral analysis, anomaly detection
**Setup**: Ensure EDR agents active, behavioral monitoring enabled

### Sysmon Events
**Required Fields**: Comprehensive system activity logging
**Evidence to Capture**: File creation, network connections, process activity
**Setup**: Deploy Sysmon with comprehensive configuration

## 7. Detection Opportunities

### Phase 1: Resource Preparation
**Candidate Detections**: Unusual account creation patterns, bulk domain registration
**ATT&CK Indicators**: T1583.001, T1583.007
**Log Fields**: Account creation timestamps, domain registration records
**False Positives**: Legitimate infrastructure setup, automated provisioning
**Tuning**: Focus on timing patterns, unusual bulk operations

### Phase 2: Initial Compromise
**Candidate Detections**: Phishing email delivery, malicious link clicks
**ATT&CK Indicators**: T1566.002
**Log Fields**: Email headers, URL clicks, process execution
**False Positives**: Legitimate email campaigns, user error
**Tuning**: Focus on sender reputation, URL reputation, user behavior baselines

### Phase 3: Payload Execution
**Candidate Detections**: PowerShell execution, script-based attacks
**ATT&CK Indicators**: T1059.001, T1204.002
**Log Fields**: Command line arguments, script content, execution context
**False Positives**: Legitimate administration, automated scripts
**Tuning**: Focus on unusual PowerShell usage, encoded commands, suspicious parent processes

### Phase 4: Persistence and Evasion
**Candidate Detections**: Registry modifications, obfuscation techniques
**ATT&CK Indicators**: T1027, T1112
**Log Fields**: Registry key access, file modifications, process behavior
**False Positives**: System updates, legitimate software installation
**Tuning**: Focus on unusual registry locations, obfuscated content patterns

### Phase 5: Command and Control
**Candidate Detections**: Unusual network traffic, C2 protocol usage
**ATT&CK Indicators**: T1071.001
**Log Fields**: HTTP requests, DNS queries, connection patterns
**False Positives**: Legitimate web traffic, software updates
**Tuning**: Focus on timing patterns, unusual user agents, data exfiltration indicators
